# UX / Accessibility Audit — Public site (2026-06-18)

Method: UI/UX Pro Max skill checklist (web-relevant subset) applied to the marketing
site (`src/components/sections/*`, `layout/Nav.tsx`, `globals.css`). No code changed.
Severity: **High** = fix before next release · **Medium** = should fix · **Low** = polish.

## Strengths (keep)
- `prefers-reduced-motion` honored in CSS **and** JS — GSAP/Lenis/canvas loops are skipped, not just dimmed. Exemplary.
- Heavy libs (GSAP, Lenis, SplitType, canvases) are `dynamic()` / lazy-imported, off the SSR/initial bundle.
- SVG icons, no emoji-as-icon. Fluid `clamp()` type scale. JSON-LD + good metadata. Touch fallback for hover-revealed menu descriptions (`@media (hover:none)`).

## Status (2026-06-18)

  override in `shop.css`; `shop.css` `outline:none` now overridden by `:focus-visible`).

## Findings

### High
1. **No visible focus indicator anywhere.** No `:focus-visible` styles in any component, and `src/app/shop/shop.css:135` sets `outline:none` on an input with no replacement. Keyboard users can't see where they are. → Add a global `:focus-visible` ring (amber on dark) and remove the bare `outline:none`.
2. **Dead controls in the nav.** The hamburger button ([Nav.tsx:36](src/components/layout/Nav.tsx:36)) and the user-icon button ([Nav.tsx:113](src/components/layout/Nav.tsx:113)) have `aria-label`s and hover states but **no `onClick`/handler** — they look interactive but do nothing. → Wire them up or make them non-interactive.
3. **Low-contrast text below 4.5:1** (warm-on-near-black):
   - Hero scroll arrow `rgba(212,160,23,0.35)` ([Hero.tsx:31](src/components/sections/Hero.tsx:31)).
   - Menu description `rgba(212,160,23,0.38)` ([Menu.tsx:64](src/components/sections/Menu.tsx:64)).
   - `--amber2 #8B6914` used for prices, JP names, captions (`Menu.tsx`, `Hero.tsx`) — ~3:1, fails for small text.
   - `--dim rgba(232,224,208,0.45)` as muted text. → Raise opacity/lightness for any text conveying real info; reserve sub-3:1 tones for purely decorative glyphs.

### Medium
4. **Body font-weight 200** ([globals.css:44](src/app/globals.css:44)) is too thin for JP body on dark — compounds the contrast problem. → Use ≥300 body, ≥400 for small/important text.
5. **Touch targets below 44px.** User icon 26×26 ([Nav.tsx:121](src/components/layout/Nav.tsx:121)), hamburger lines, `shop/login` 52px-wide-but-short ([Nav.tsx:72](src/components/layout/Nav.tsx:72)). → Expand hit areas (padding / `hitSlop`-equivalent) to ≥44px.
6. **Heading hierarchy.** Only the Hero `<h1>` exists; sections (Menu, Story, etc.) label themselves with `<span>`, so there are no `<h2>`s and menu item names aren't headings. → Add semantic section headings for screen-reader/SEO structure.

### Low
7. **Always-on canvas load.** Stars + Smoke + SmokeCanvas full-screen layers run continuously (battery/main-thread). Acceptable today, but treat as the ceiling — don't add more. Consider pausing canvases when the tab/section is offscreen.
8. **Decorative watermark** "menu" is `aria-hidden` ✓ — good; keep applying that to future decorative text.

## Suggested fix order
1 (focus ring) → 2 (dead controls) → 3 (contrast) → 4 (body weight) → 5 (touch targets) → 6 (headings) → 7–8 (polish).
Items 1–4 are small, high-impact CSS/markup edits.
